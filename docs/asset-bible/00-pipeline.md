# 00 — Пайплайн: Geometry Nodes + рендер

## Цели и контекст

Нужны большие наборы устройств/кораблей/зданий, стилистически единые внутри расы
и явно различимые между расами. Ручное моделирование не масштабируется на N рас ×
M устройств × вариации. Решение — процедурная генерация: архетип устройства ×
стилевой кит расы × seed.

## Архитектура нод-групп

```
GN_Device_Master  (единственный модификатор на объекте)
│
├── вход: Device Type   (menu: engine / generator / shield / weapon_beam /
│                              weapon_launcher / scanner / special)
├── вход: Race Style    (menu: humans / shuffie / ...)
├── вход: Seed          (int)
├── вход: Size Class    (int 1–3: small / medium / large)
├── вход: Detail Level  (float 0–1, для LOD и иконок)
│
├── GN_Arch_Engine ─┐
├── GN_Arch_Shield ─┤   архетипы: функциональная БАЗА устройства —
├── GN_Arch_...    ─┘   силуэт, пропорции, mount points, слоты под детали
│
└── GN_Style_Humans ─┐  стилевые киты: обрабатывают базу —
    GN_Style_Shuffie ┘  панелизация, гриблы, фаски, материалы, палитра
```

### Контракт «архетип → стилевой кит»

Архетип отдаёт киту не просто меш, а меш **с атрибутами**, по которым кит
понимает, что где находится:

| Атрибут | Тип | Смысл |
|---|---|---|
| `part_id` | int | Крупная функциональная зона (корпус=0, сопло=1, эмиттер=2 …) |
| `greeble_slot` | bool (face) | Грань, на которую кит может ставить детали |
| `mount_point` | bool (point) | Точка крепления к кораблю/зданию |
| `panel_density` | float (face) | Рекомендуемая плотность панелизации 0–1 |

Стилевой кит **обязан** реализовать одинаковый интерфейс (см. `races/_template.md`),
тогда любая новая раса подключается без изменения архетипов.

### Naming conventions

| Сущность | Шаблон | Пример |
|---|---|---|
| Мастер-группа | `GN_Device_Master` | — |
| Архетип | `GN_Arch_<Type>` | `GN_Arch_Engine` |
| Стилевой кит | `GN_Style_<Race>` | `GN_Style_Shuffie` |
| Вспомогательная группа кита | `GN_Style_<Race>_<Fn>` | `GN_Style_Humans_Panels` |
| Материал | `MAT_<race>_<role>` | `MAT_humans_hull` |
| Объект-устройство | `DEV_<race>_<type>_s<seed>` | `DEV_shuffie_shield_s004` |
| Рендер | `<race>/<type>_s<seed>_<size>.png` | `humans/engine_s003_m.png` |

## Рендер-пайплайн (изображения устройств)

Единая сцена для всех рас, чтобы наборы были сравнимы:

- **Камера:** одна ортографическая ¾-ракурс (иконки) + опционально турнтейбл.
- **Свет:** трёхточечный риг, фиксированный; расовая палитра живёт в материалах,
  не в свете.
- **Фон:** прозрачный (`film_transparent = True`), PNG RGBA.
- **Разрешения:** 512×512 (UI-иконки), 1024×1024 (Asset Bible / маркетинг).
- **Движок:** EEVEE Next для массового прогона; Cycles для hero-рендеров.

Batch-прогон: `tools/render_device_set.py` перебирает `Device Type × Seed`
для заданной расы и складывает PNG по конвенции имён выше. Итог одного прогона —
контрольный лист: 7 типов × 8 сидов = 56 изображений на расу.

## LOD и производительность

- `Detail Level = 1.0` — рендер иконок и hero-shots;
- `Detail Level ≈ 0.4` — realtime-модель в игре (гриблы прореживаются
  по атрибуту `panel_density`);
- Realize Instances — только на экспорте; внутри графа держать инстансы.

## Definition of Done для набора устройств расы

- [ ] Все 7 архетипов рендерятся с китом расы без ручных правок меша
- [ ] 8 сидов на архетип дают различимые, но узнаваемые вариации
- [ ] Side-by-side с другой расой: расу видно по силуэту за < 1 секунды
- [ ] Контрольный лист приложен к `races/<race>.md`
