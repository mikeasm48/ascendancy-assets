# Референсы устройств — политика папки

> **Оригиналы:** полноразмерные версии всех референсов лежат вне
> репозитория — `~/java/ascendancy-refs-originals/devices/`. Здесь
> хранятся ужатые копии (изображения ≤1024px, JPEG q85; GLB заменены
> `*_preview.png`) — их достаточно для сверки при доработке рецептов
> `tools/device_recipes_*.py`. Для генерации GLB-наборов эта папка
> не требуется вовсе (см. 02-device-set.md).

Раскладка — по расам/наборам:

- `core/` — набор по умолчанию, стилизация под иконки Ascendancy 1995.
  Формат имени файла: `<Тип>_core_<Название>_<версия>.jpg`, например
  `Engine_core_TonklinMotor_1.jpg`. Типы: Engine, StarLaneDrive,
  Generator, Scanner, Shield, Weapon, Aux. Одна картинка = одна модель
  в `device_constructor.glb` (узел `<Тип>_<Название>`); поколений и
  привязки к спецификациям ascendancy-remake на этом уровне нет.
- `humans/` — по категориям устройств (`engine/`, `generator/`,
  `shield/`, `scanner/`, `star_lane_drive/`, `weapon/`, `aux/`);
  набор humans следует `DefaultTechCatalog.java` с поколениями.

## Формат референсов

- **Лучше всего:** плоские изображения PNG/JPG — рендеры, концепты,
  скриншоты, наброски от руки. Вид ¾ предпочтителен.
- Тоже подходит: 3D-модель (GLB/OBJ) — в репо остаётся `*_preview.png`.
- Текстом — как дополнение к картинке («сопло больше», «двойной ствол»).

Новая версия того же устройства — инкремент суффикса `_<версия>`;
неактуальные версии удаляются из репо (оригиналы остаются в
ascendancy-refs-originals).
