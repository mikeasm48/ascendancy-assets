# Ascendancy Neo — UI style

Единый источник правды по визуальному языку 2D-интерфейса (HUD, меню,
диалоги, панели). Отображаемое название игры — **Ascendancy Neo**.
Референсы — 14 скриншотов основных экранов в
[`refs/ui/Ascendancy_Neo_Screens/`](refs/ui/Ascendancy_Neo_Screens/) —
как и остальные refs, это ужатые копии (jpeg q85, ≤1024px);
полноразмерные PNG-оригиналы лежат вне git в
`~/java/ascendancy-refs-originals/ui/Ascendancy_Neo_Screens/`.

## Assets vs код — и почему

**Весь визуал интерфейса — код (векторная отрисовка), кроме шрифта.**

В `ascendancy-remake` уже сегодня, до этой задачи, 100% HUD/UI рисуется
процедурно: `StarfieldBackground` (звёздный фон), `HudOverlay` (рамки,
тултипы, D-pad, zoom), `HudIcons` (иконки-глифы: песочные часы, стрелки,
+/-, оружейный значок) — всё через `ShapeRenderer` и векторные примитивы,
без единой UI-текстуры. Текстурные ассеты в проекте существуют только для
3D-моделей (устройства/здания/корабли, тот самый пайплайн из
`00-pipeline.md`–`03-building-set.md`) и планетных текстур.

Стиль Neo (шеврон-рамки, скруглённые/срезанные углы, greeble-засечки,
неоновое свечение, D-pad, статус-бары) продолжает эту же логику: все
формы — параметризуемая геометрия, отрисовывается кодом на любом
разрешении без артефактов растяжения, свечение — имитация слоем
полупрозрачных обводок (см. `NEO_GLOW_LAYERS` ниже), а не bloom-шейдер.
Единственный элемент, который нельзя нарисовать примитивами
`ShapeRenderer`, — начертание шрифта тайтлов и текста. Поэтому единственный
новый бинарный ассет для UI — **два файла шрифтов** (см. «Типографика»).

Это сознательное продолжение существующей архитектуры, а не новое
решение с нуля — при добавлении новых экранов в стиле Neo эту границу
менять не нужно: новый визуальный элемент почти всегда means новая
процедурная фигура в `view/screen/neo/`, не текстура.

## Цветовые токены

Значения сведены по всем 14 референсам (усреднение по повторяющимся
элементам — рамки, заголовки, интерактивные кнопки).

| Токен | Hex | RGBA (0..1) | Использование |
|---|---|---|---|
| `NEO_BG` | `#0A0806` | 0.04, 0.03, 0.02, 1 | Фон панелей/экрана (почти чёрный, тёплый оттенок) |
| `NEO_GOLD` | `#E8A33D` | 0.91, 0.64, 0.24, 1 | Рамки панелей, заголовки, статичные лейблы |
| `NEO_GOLD_BRIGHT` | `#F5C878` | 0.96, 0.78, 0.47, 1 | Ядро свечения рамок/тайтлов (внутренняя, самая яркая обводка) |
| `NEO_GOLD_DIM` | `#8A6438` | 0.54, 0.39, 0.22, 1 | Второстепенный текст, разделители, неактивные элементы |
| `NEO_CYAN` | `#4FD3FF` | 0.31, 0.83, 1.00, 1 | Интерактивные элементы: кнопки, иконки, D-pad, значения статов, поле ввода |
| `NEO_CYAN_BRIGHT` | `#B9F0FF` | 0.73, 0.94, 1.00, 1 | Ядро свечения активных/hover интерактивных элементов |
| `NEO_CYAN_DIM` | `#2E7A94` | 0.18, 0.48, 0.58, 1 | Неактивная (enabled, но не hover) обводка интерактивных элементов |
| `NEO_DANGER` | `#E8542A` | 0.91, 0.33, 0.16, 1 | Опасные/необратимые действия (кнопка "Abandon", предупреждающий треугольник) |
| `NEO_STAT_HULL` | `#6FBF4F` | 0.44, 0.75, 0.31, 1 | Полоса прочности корпуса (зелёная) |
| `NEO_STAT_ENERGY` | `#4FA8E8` | 0.31, 0.66, 0.91, 1 | Полоса энергии/щитов (синяя) |
| `NEO_STAT_SECONDARY` | `#D84FC0` | 0.85, 0.31, 0.75, 1 | Вторичный стат на панели корабля (пурпурная полоса) |
| `NEO_STAT_INDUSTRY` | `#E8A33D` | = `NEO_GOLD` | Полоса индустрии/прогресса (тот же тон, что рамки) |

Заполнение (`fill`) закрашенных элементов (панели, поля ввода) — всегда
`NEO_BG` с alpha 0.80–0.92, не полностью непрозрачный: звёздный фон должен
слабо просвечивать сквозь панели (видно на `Neo_GalaxyMap.jpg`).

## Геометрия

- **Шеврон-рамка панели**: прямоугольник со срезанными (chamfered) углами
  под ~45°, срез ≈ 3% от меньшей стороны панели. Двойная обводка: внешняя
  тонкая (1px, `NEO_GOLD_DIM`) + внутренняя основная (2px, `NEO_GOLD`) с
  зазором 2–3px между ними — это и даёт эффект "контурного свечения" без
  шейдера.
- **Greeble-засечки**: короткие (4–8px) перпендикулярные штрихи или
  группы точек (`....`) на рамке — у верхней кромки по центру, у одного из
  внешних углов, иногда посередине боковой кромки. Несимметричны и не
  повторяются буквально панель к панели — генерируются по seed панели
  (позиция/id), не хардкодятся один-в-один.
- **Ромб-глиф**: маленький повёрнутый на 45° квадрат (ромб) по центру
  нижней кромки крупных полноэкранных рамок (Galaxy Map, Planet Screen,
  Star System Screen) — декоративный якорь, не интерактивен.
- **Кнопки/поля списка**: тот же принцип среза углов, но меньший радиус
  среза (~15–20% высоты кнопки) — почти "восьмиугольная пилюля". Обычные
  кнопки/лейблы — `NEO_GOLD` обводка; интерактивные/выбираемые (список
  рас, список сохранений, строки кораблей) — `NEO_CYAN` обводка.
  Первичное действие в паре кнопок (OK) — `NEO_CYAN`; вторичное/нейтральное
  (Cancel) — `NEO_GOLD`; опасное (Abandon) — `NEO_DANGER`.
- **D-pad / zoom-кластер**: сетка 3×3 (или "+"-крест 3×3 с угловыми
  пустыми ячейками не занятыми, как на `Neo-3D_Widget.jpg`) квадратных
  кнопок со срезанными углами, зазор между кнопками 3–4px, вся сетка в
  общей рамке `NEO_GOLD` с маленькими точками-заклёпками в 4 внешних
  углах. Центральная кнопка — сброс вида (кружок `○`), остальные —
  направление/зум (стрелки, `+`/`-`).
- **Статус-бар**: тонкая рамка `NEO_GOLD_DIM`, заполнение — сплошной цвет
  токена стата на всю высоту, ширина заполнения = доля от максимума.
  Числовое значение — по центру или под баром, `NEO_CYAN` если "живой"
  статистический параметр (Science/Industry/Ecology на PlanetScreen),
  `NEO_GOLD` если статичный лейбл.
- **Сворачиваемая нижняя/верхняя панель** (лента событий): прикреплена к
  краю экрана, потянуть за вкладку с шевроном (▼ снизу открывает вниз, ▲
  сверху открывает вверх) по центру общей кромки. Строки списка —
  разделены тонкой линией `NEO_GOLD_DIM`, без отдельных рамок на каждую
  строку.

## Типографика

Два начертания, разное назначение — в референсах чётко видно, что
заголовки геометричны и растянуты по трекингу, а весь остальной текст —
округлый и компактный, это не одно и то же семейство.

| Роль | Шрифт | Где | Регистр/трекинг |
|---|---|---|---|
| Display / заголовки | **Orbitron** (SIL OFL 1.1) | "ASCENDANCY", "GALAXY MAP", "SHIPS", "DAY 47", заголовки панелей | ВСЕ ЗАГЛАВНЫЕ, letter-spacing ≈ +0.15em |
| Body / списки / лейблы | **Quicksand** (SIL OFL 1.1) | подписи, строки списков, кнопки, числа статов, подсказки внизу экрана | как есть (Title Case/обычный), обычный трекинг |

Файлы (straight-copy ассет, **не** часть релизного пайплайна
`~/.ascendancy/assets` — см. ниже), взяты из google/fonts (SIL OFL 1.1):
`refs/ui/fonts/Orbitron-Bold.ttf` + `Orbitron-OFL.txt`,
`refs/ui/fonts/Quicksand-Medium.ttf` + `Quicksand-OFL.txt`. Оба — variable
fonts (ось `wght`); `ascendancy-remake` рендерит именованный инстанс
Bold/Medium через `gdx-freetype` (см. Part B в плане реализации).

`ascendancy-remake` использует `gdx-freetype` для растеризации `.ttf` в
`BitmapFont` во время выполнения (нужный размер/цвет на лету), поэтому
хранится именно исходный `.ttf`, а не готовый bitmap-атлас.

**Почему не через `~/.ascendancy/assets` / GitHub Release:** этот пайплайн
версionирует тяжёлый контент, специфичный для расы (устройства/здания),
который игрок докачивает по `AssetDownloader`. Шрифт — маленький,
статичный, нужен ещё до старта любой загрузки (заставка/меню), поэтому
копируется прямо в `ascendancy-remake/src/main/resources/ui/fonts/` и
коммитится в игровой репозиторий как обычный ресурс.
