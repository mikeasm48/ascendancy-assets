# Референсы устройств — что сюда класть

> **Оригиналы:** полноразмерные версии всех референсов (включая исходные
> GLB-модели) лежат вне репозитория — `~/java/ascendancy-refs-originals/devices/`.
> Здесь хранятся ужатые копии (изображения ≤1024px, GLB заменены
> `*_preview.png`) — их достаточно для сверки при доработке рецептов
> `tools/device_recipes_*.py`. Для генерации GLB-наборов эта папка
> не требуется вовсе (см. 02-device-set.md).

Папка для примеров внешнего вида устройств. По ним перестраиваются
архетипы `GN_Arch_*` (силуэты), стилевые киты остаются как есть.

## Минимальный набор — 7 картинок, по одной на тип

| Файл | Тип устройства | Что должно читаться |
|---|---|---|
| `engine.png` | двигатель | «толкает» — корпус + сопло |
| `generator.png` | генератор | «излучает» — ядро + радиаторы/контур |
| `shield.png` | щит | «оборачивает» — эмиттер + дуга/кольцо |
| `weapon_beam.png` | лучевое оружие | «целится» — ствол/фокусатор вдоль оси |
| `weapon_launcher.png` | залповое оружие | «выпускает залпом» — кассета/барабан |
| `scanner.png` | сканер | «слушает» — тарелка/антенна |
| `special.png` | спец-устройства | нестандартный силуэт |

## Формат

- **Лучше всего:** плоские изображения PNG/JPG — скриншоты из референсных
  игр, концепты, грубые наброски от руки. Вид сбоку или ¾.
- Тоже подходит: 3D-модель (GLB/OBJ) — разберу как исходники shipyard.
- Текстом — как дополнение к картинке («сопло больше», «двойной ствол»).

## Опционально

- Если у рас силуэты должны отличаться не только материалами/деталями —
  два файла: `engine_humans.png`, `engine_shuffie.png` и т.д.
- Отдельные референсы уровней («тот же двигатель, но круче») — суффикс
  `_l1`/`_l3`/`_l5`. По умолчанию прогресс уровня = размер + плотность
  деталей (правило 3 из 01-device-taxonomy.md).
- Референсы орбитальных структур (доки!) — `dock.png`.

Картинки можно просто прислать в чат — я сам разложу их сюда.
